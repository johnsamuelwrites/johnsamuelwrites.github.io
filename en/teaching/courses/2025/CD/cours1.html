Approaches

  • Supervised Learning: The model is trained on a labeled dataset where input examples are associated with desired outputs. The model learns to make predictions on new data based on these associations.
  • Unsupervised Learning: The model is exposed to unlabeled data and seeks to discover patterns, structures or intrinsic relationships in the data.
  • Semi-supervised Learning: A combination of the two preceding approaches, using both labeled and unlabeled data for training.
  • Reinforcement Learning: The model learns to make decisions by interacting with its environment. It receives rewards or penalties based on its actions, which guides its learning.

In the context of machine learning classification, evaluating a model's performance involves understanding the different types of predictions it can make relative to reality. True positives (TP) and true negatives (TN) are two of these elements.

  • True Positives (TP): True positives represent cases where the model correctly predicts the positive class. In other words, it has correctly identified examples that actually belong to the class the model is trying to predict.
  • True Negatives (TN): True negatives represent cases where the model correctly predicts the negative class. This means it has correctly identified examples that do not belong to the class the model is trying to predict.

2.2. Classification Methods

Evaluation

The F1-score is the harmonic mean of precision and recall. It provides an overall measure of a classification model's performance, taking both precision and recall into account. It is particularly useful when classes are imbalanced.

  • F1-score \[f1 = 2 * \frac{(p * r)}{(p + r)}\]
  • F1-score: best value at 1 (perfect precision and recall) and worst at 0.

The F1-score accounts for both Type I errors (false positives) and Type II errors (false negatives), thus providing a balanced measure of model performance.

    2.2.3 Regression

    Linear Regression

    Linear regression is a mathematical model that represents a linear relationship between an independent variable \(x_i\) and a dependent variable \(y_i\). The model takes the form of a straight line (for simple linear regression) or a parabola (for multiple linear regression).

    • Straight line: \(y_i = β_0 + β_1x_i + ε_i\) where \(β_0\) and \(β_1\) are the regression coefficients, \(x_i\) is the independent variable, and \(ε_i\) is the residual error.
    • Parabola: \(y_i = β_0 + β_1x_i + β_2x_i^2 +ε_i\) where \(β_0\), \(β_1\), and \(β_2\) are the regression coefficients for each term, \(x_i\) is the independent variable, and \(ε_i\) is the residual error.

    Straight line: \(y_i = β_0 + β_1x_i + ε_i\) where \(β_0\) and \(β_1\) are the regression coefficients, \(x_i\) is the independent variable, and \(ε_i\) is the residual error.

    2.2.5. Association Rules

    Formal Definition

    • Let \(I\) be a set of \(n\) binary attributes called items
    • Let \(T\) be a set of \(m\) transactions called a database
    • Let \(I\) = \(\{(i_1,...,i_n)\}\) and \(T\) = \({(t_1,...,t_m)}\)

      • \(X ⇒ Y\), where \(X ⇒ Y ⊆ I\)

    2.3.4. Naive Bayes Classification

    Bayes' Theorem

    • \(P(A), P(B)\) are the probabilities of observing A and B independently of each other.
    • \(P(A|B)\) is a conditional probability, the probability that event \(A\) occurs given that \(B\) is true
    • \(P(B|A)\) is a conditional probability, the probability that event \(B\) occurs given that \(A\) is true
    • \(P(B) ≠ 0\)

    \[P(A|B) = \frac{(P(B|A).P(A))}{P(B)}\]

    Bayes' Theorem: Message Classification

    • \(P(S)\) is the overall probability that a given message is spam.
    • \(P(H)\) is the overall probability that a given message is not spam.
    • \(P(S|W)\) is the probability that a message is spam, given that the word appears in it;
    • \(P(W|S)\) is the probability that the word appears in spam messages;
    • \(P(W|H)\) is the probability that the word appears in ham messages.

    \[P(S|W) = \frac{P(W|S) \cdot P(S)}{P(W|S) \cdot P(S) + P(W|H) \cdot P(H)}\]

    3. ML Workflow for Scientific Problems

    Applying machine learning to scientific problems follows a structured workflow that guides the practitioner from raw data to a deployable model.

    1. Problem definition: Formulate the scientific question as an ML task (classification, regression, anomaly detection, clustering, etc.).
    2. Data collection and exploration: Gather domain data and perform exploratory data analysis (EDA) to understand distributions, correlations, and data quality.
    3. Data preparation: Clean, transform, and engineer features to produce a suitable feature matrix.
    4. Model selection and training: Choose candidate algorithms and train them on a designated training set.
    5. Evaluation and validation: Assess model performance on held-out data using appropriate metrics.
    6. Iteration: Refine the model, features, or data collection strategy based on evaluation results.
    7. Deployment and monitoring: Integrate the model into a production environment and monitor its performance over time.

    3.1. Problem Formulation

    3.1. Problem Formulation

    Key Considerations

    • Data leakage: Occurs when information from the test set (or future data) is inadvertently used during training. It leads to overly optimistic evaluation results and poor generalization in production.
      • Common causes: scaling/normalizing with statistics computed on the full dataset, including target-derived features, temporal data split errors.
      • Prevention: fit all preprocessing transformers only on training data and apply them to validation/test data without refitting.
    • Class imbalance: When one class is much more frequent than others, standard accuracy can be misleading. Strategies include oversampling (SMOTE), undersampling, cost-sensitive learning, and appropriate metrics (F1, AUC-ROC).
    • Target definition: Ambiguity in defining the prediction target can undermine the entire workflow; the target variable must reflect the scientific question precisely.

    3.2. Train / Validation / Test Partitions

    Proper data partitioning is essential to obtain unbiased estimates of model performance and to avoid overfitting.

    • Training set: Used to fit the model parameters. Typically 60–80% of the data.
    • Validation set: Used to tune hyperparameters and select among competing models. Kept separate from the test set to avoid selection bias.
    • Test set: Used only once for the final evaluation of the chosen model. Must remain untouched during model development. Typically 10–20% of the data.
    • Stratification: When splitting, stratified sampling ensures each partition preserves the original class distribution, which is especially important under class imbalance.
    • Temporal splits: For time-series data, splits must respect chronological order to prevent future data leaking into the training set.

    3.2. Cross-Validation

    Cross-validation is a resampling technique used to evaluate a model's generalization performance when the dataset is too small for a separate validation set.

    • k-Fold Cross-Validation: The dataset is divided into k equally sized folds. The model is trained on k-1 folds and evaluated on the remaining fold; this process is repeated k times. The final performance estimate is the average over the k runs. Common choices: k = 5 or k = 10.
    • Stratified k-Fold: Each fold preserves the class distribution, recommended for classification tasks with imbalanced classes.
    • Leave-One-Out (LOO): A special case of k-fold where k equals the number of samples. Provides an almost unbiased estimate but is computationally expensive for large datasets.
    • Nested Cross-Validation: An outer loop estimates generalization performance while an inner loop tunes hyperparameters, providing unbiased model selection.

    3.3. Data Preparation

    Raw data rarely satisfies the assumptions of ML algorithms. Data preparation transforms the raw feature matrix into a form suitable for learning.

    • Feature scaling: Many algorithms (SVM, k-NN, gradient descent-based methods) are sensitive to the scale of features.

      • Min-Max normalization: \(x' = \frac{x - x_{min}}{x_{max} - x_{min}}\). Scales features to the interval [0, 1].
      • Robust scaling: Uses the median and interquartile range, robust to outliers.
    • Categorical encoding: Convert categorical variables to numerical representations (one-hot encoding, label encoding, target encoding).

    3.3. Data Preparation

    Missing Data

    • Mechanisms of missingness:
      • MCAR (Missing Completely At Random): absence is unrelated to the data.
      • MAR (Missing At Random): absence depends on observed variables.

    3.4. Clustering Algorithms

    Clustering groups unlabeled observations so that objects in the same cluster are more similar to each other than to those in other clusters.

    • K-Means: Partitions data into k clusters by minimizing within-cluster sum of squared distances.
      • Requires specifying k in advance.
      • Sensitive to initialization; the k-means++ strategy improves convergence.
      • Assumes spherical, equally sized clusters; struggles with non-convex shapes.
    • Hierarchical Clustering: Builds a dendrogram representing nested cluster structures.
      • Agglomerative (bottom-up): starts with each point as its own cluster and merges the closest pair iteratively.
      • Divisive (top-down): starts with one cluster and recursively splits.
      • Linkage criteria: single, complete, average, Ward.

    3.4. Clustering Algorithms

    DBSCAN

    DBSCAN (Density-Based Spatial Clustering of Applications with Noise) discovers clusters of arbitrary shape based on local density.

    • Core points: A point is a core point if at least MinPts points lie within distance \(\varepsilon\) of it.
    • Border points: Reachable from a core point but not core points themselves.
    • Noise points: Neither core nor border points; treated as outliers.
    • Advantages:
      • Does not require specifying the number of clusters in advance.
      • Robust to outliers (noise points).
      • Discovers clusters of arbitrary shape.
    • Limitations: Sensitive to the choice of \(\varepsilon\) and MinPts; struggles with varying-density clusters and high-dimensional data.

    3.4. Clustering Algorithms

    Cluster Evaluation

    • Internal metrics (no ground-truth labels required):
      • Silhouette coefficient: measures how similar a point is to its own cluster compared to other clusters. Ranges from -1 to 1; higher is better.
      • Davies-Bouldin index: average similarity ratio of each cluster with its most similar cluster; lower is better.
      • Calinski-Harabasz index: ratio of between-cluster dispersion to within-cluster dispersion; higher is better.
    • External metrics (require ground-truth labels):

    • Choosing k: The elbow method plots within-cluster inertia against k; the optimal k is at the "elbow" of the curve.

    3.5. Anomaly Detection Algorithms

    Anomaly detection algorithms identify rare observations that differ significantly from the majority of the data.

    • One-Class SVM (OC-SVM): A kernel-based method that learns a decision boundary enclosing the majority of training data in a high-dimensional feature space.
      • Trained on normal (non-anomalous) data only.
      • The parameter \(\nu \in (0,1]\) controls the fraction of training samples allowed to be outliers.
      • At test time, points outside the learned boundary are classified as anomalies.
      • Effective in high-dimensional spaces but sensitive to the choice of kernel and hyperparameters.

    3.5. Anomaly Detection Algorithms

    Isolation Forest

    Isolation Forest is an ensemble-based anomaly detector that exploits the fact that anomalies are few and different, making them easier to isolate.

    • Principle: An isolation tree is built by recursively partitioning the data with random splits. Anomalous points, being sparse and extreme, require fewer splits to be isolated.
    • Anomaly score: The average path length across an ensemble of isolation trees to isolate a point. Short average path length indicates an anomaly.
    • Advantages:
      • Linear time complexity; scales well to large datasets.
      • Does not require a distance or density metric.
      • Handles high-dimensional data effectively.
    • Hyperparameter: contamination — the expected proportion of anomalies in the data, used to set the decision threshold.
